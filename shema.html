Database Schema: Book Ticket System

Table: users

Column Type Constraints Description
id BIGINT PRIMARY KEY, AUTO_INCREMENT Unique identifier for each user.
username VARCHAR NOT NULL, UNIQUE The user's login name.
password VARCHAR NOT NULL The user's encrypted password.
email VARCHAR NOT NULL The user's email address.
Relationships: Has a one-to-many relationship with the bookings table.

Table: movies

Column Type Constraints Description
id BIGINT PRIMARY KEY, AUTO_INCREMENT Unique identifier for each movie.
name VARCHAR NOT NULL The title of the movie.
genre VARCHAR The genre of the movie (e.g., Action, Comedy).
description TEXT A brief summary of the movie.
duration INT The length of the movie in minutes.
releaseDate DATE The official release date of the movie.
language VARCHAR The language the movie is in.
Relationships: Has a one-to-many relationship with the show_entity table.

Table: theaters

Column Type Constraints Description
id BIGINT PRIMARY KEY, AUTO_INCREMENT Unique identifier for each theater.
theaterName VARCHAR NOT NULL The name of the theater.
theaterLocation VARCHAR NOT NULL The physical location of the theater.
theaterCapacity INT NOT NULL The total number of seats in the theater.
theaterScreenType VARCHAR The type of screen (e.g., IMAX, 3D).
Relationships: Has a one-to-many relationship with the show_entity table.

Table: show_entity

Column Type Constraints Description
id BIGINT PRIMARY KEY, AUTO_INCREMENT Unique identifier for each show.
showTime DATETIME NOT NULL The date and time the show starts.
price DOUBLE NOT NULL The price of a single ticket for this show.
movie_id BIGINT FOREIGN KEY to movies(id) Links to the movie being shown.
theater_id BIGINT FOREIGN KEY to theaters(id) Links to the theater where the show is held.
Relationships: Has many-to-one relationships with movies and theaters, and a one-to-many relationship with bookings.

Table: bookings

Column Type Constraints Description
id BIGINT PRIMARY KEY, AUTO_INCREMENT Unique identifier for each booking.
numberOfSeats INT NOT NULL The number of seats booked.
bookingTime DATETIME NOT NULL The timestamp when the booking was made.
price DOUBLE NOT NULL The total price for the booking.
bookingStatus ENUM ('CONFIRMED', 'CANCELLED', 'PENDING') The current status of the booking.
user_id BIGINT FOREIGN KEY to users(id) Links to the user who made the booking.
show_id BIGINT FOREIGN KEY to show_entity(id) Links to the specific show that was booked.
Relationships: Has many-to-one relationships with users and show_entity.

Table: booking_seat_numbers

Column Type Constraints Description
booking_id BIGINT FOREIGN KEY to bookings(id) Links back to the parent booking.
seatNumbers VARCHAR Stores a single seat number (e.g., "A1", "B5").
Note: This is a collection table created by JPA to store the list of seat numbers for a booking.
